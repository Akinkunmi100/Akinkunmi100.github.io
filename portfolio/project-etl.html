Data Engineering

High-Throughput ETL Pipeline

Architected a fault-tolerant Python/SQL pipeline processing 10M+ daily records, reducing data availability latency from 24 hours to 15 minutes.

The Challenge

The company's data warehouse was updated via a fragile, nightly batch script. When it failed, the business team lost a whole day of insights. As data volume grew to 10M rows/day, the nightly window was no longer sufficient.

The Solution

I re-architected the ingestion layer to move from Batch to Near-Real-Time processing.

  • Built modular extraction scripts using Python and Pandas.
  • Orchestrated workflows using Apache Airflow to handle dependencies and retries.
  • Optimized SQL transformations using window functions and proper indexing.

The Results

15minLatency
10M+Rows/Day
99.9%Uptime
